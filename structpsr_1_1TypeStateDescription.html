Detailed Description

template<typename StateTy>
struct psr::TypeStateDescription< StateTy >

Interface for a type state problem to be used with the IDETypeStateAnalysis. It needs to provide a finite state machine to handle state changes and a list of functions that are relevant to the analyzed api. Factory functions are those functions which generate objects/variables of target type, e.g. file handles generated by fopen(). Consuming functions operate on objects/variables of target type, e.g. fwrite(). Functions that are both factory and consumer, e.g. freopen(), cannot be modeled by this interface, since analyzing such functions correctly introduces dependencies that cannot be solved by a distributive framework such as IDE.

See also
CSTDFILEIOTypeStateDescription as an example of type state description.
